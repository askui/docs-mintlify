## Prerequisites

<AccordionGroup>
<Accordion title="Meet Your Terminal" icon="terminal">

A terminal is a text-based way to talk directly to your computer. Instead of clicking through folders and menus, you type short commands and press Enter. It does the same things — opens folders, creates files, runs programs — but faster and more precisely.

### Open your terminal

<Tabs>
  <Tab title="Windows">
    Press the `Windows` key on your keyboard. Type `PowerShell`. Click on **Windows PowerShell** when it appears. A dark window with a blinking cursor will open — that's your terminal.
  </Tab>
  <Tab title="macOS">
    Press `Cmd + Space` to open Spotlight. Type `Terminal` and press Enter. A window with a blinking cursor will open — that's your terminal.
  </Tab>
  <Tab title="Linux">
    Press `Ctrl + Alt + T`. A window with a blinking cursor will open — that's your terminal. If that shortcut doesn't work, search for "Terminal" in your application menu.
  </Tab>
</Tabs>

### The 6 commands you need

These six commands work on all three operating systems (Windows PowerShell, macOS, and Linux). They're all you need for this guide:

| Command | What it does | Analogy |
| --- | --- | --- |
| `pwd` | Prints the folder you're currently in — your location on the computer | Like checking the address of the building you're standing in |
| `ls` | Lists every file and folder in your current location | Like looking around a room to see what's there |
| `cd foldername` | Moves into a folder | Like walking through a door into another room |
| `cd ..` | Moves back up one folder | Like stepping back out the way you came in |
| `mkdir name` | Creates a new folder with the name you specify | Like building a new room |
| `cp a b` | Copies file `a` to a new file `b` | Like photocopying a document |

<Note>
**Note for Windows users:** Windows PowerShell supports all the commands above. If you ever see a guide elsewhere that says Windows uses `dir` instead of `ls`, or `copy` instead of `cp` — those are the older Command Prompt equivalents. In PowerShell, both versions work. We'll stick with the universal commands in this guide.
</Note>

### Try it — create your working folder

You're going to create a folder on your Desktop where you'll keep the AskUI project. Type these commands one after another, pressing Enter after each:

```bash
cd ~/Desktop
```

```bash
mkdir askui-projects
```

```bash
cd askui-projects
```

<Check>
Now type `pwd` and press Enter. You should see a path ending in `askui-projects`. If you do, you're in the right place.
</Check>

<Warning>
**Windows note:** If `cd ~/Desktop` doesn't work in your PowerShell, use `cd $HOME\Desktop` instead.
</Warning>

</Accordion>

<Accordion title="Install Git & Python" icon="download">

You need two things installed before you can set up AskUI:

- **Python** — the programming language AskUI is built in.
- **Git** — a tool for downloading code projects from the internet.

You'll install both using your operating system's **package manager** — a tool that lets you install software by typing one command.

<Tabs>
  <Tab title="Windows">
    Windows 10 and 11 include a built-in package manager called **winget**. Copy and paste these commands into PowerShell, pressing Enter after each:

    ```bash
    winget install -e --id Python.Python.3.12
    ```

    ```bash
    winget install -e --id Git.Git
    ```

    <Warning>
    **Close PowerShell and reopen it** after installing — Windows needs to refresh its paths so it recognizes the new tools. Then navigate back:

    ```bash
    cd ~/Desktop/askui-projects
    ```
    </Warning>

    ```bash
    python --version
    ```

    ```bash
    git --version
    ```

    <Check>
    You should see version numbers — `Python 3.12.x` and `git version 2.x.x`. If either says "not recognized", close PowerShell, reopen it, and try again.
    </Check>
  </Tab>
  <Tab title="macOS">
    You'll use **Homebrew** — the most popular package manager for macOS. Copy and paste this entire line into your terminal and press Enter to install Homebrew:

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    When it asks for your password, type your Mac login password. **You won't see any characters appear as you type — that's a security feature, not a bug.** Just type and press Enter. Follow any "Next steps" it prints when done.

    ```bash
    brew install python git
    ```

    ```bash
    python3 --version
    ```

    ```bash
    git --version
    ```

    <Check>
    You should see version numbers — `Python 3.12.x` and `git version 2.x.x`. If either says "command not found", close your terminal, reopen it, and try again.
    </Check>
  </Tab>
  <Tab title="Linux">
    ```bash
    sudo apt update && sudo apt install python3 python3-pip git -y
    ```

    ```bash
    python3 --version
    ```

    ```bash
    git --version
    ```

    <Check>
    You should see version numbers — `Python 3.12.x` and `git version 2.x.x`.
    </Check>
  </Tab>
</Tabs>

<Info>
**Concept:** A programming language and version control — this combination is the universal starter kit for any software or AI project, not just AskUI. You'll encounter these same tools in web development, data science, machine learning, and DevOps.
</Info>

## 1.3 Create Your AskUI Account

You need an AskUI account before anything else — the hub at **hub.askui.com** is where you'll download the Suite installer, create your access credentials, and follow the official onboarding wizard. It's free to sign up, no credit card required.

<Steps>
  <Step title="Sign up">
    Open your browser and go to [hub.askui.com](https://hub.askui.com). Click **Sign Up** and create your account.
  </Step>
  <Step title="Open the Getting Started wizard">
    After you log in, click **Getting Started** in the left sidebar. You'll see a 4-step wizard — keep this tab open. The next three sections of this guide walk through each wizard step with more detail and explanation.
  </Step>
</Steps>

<Info>
The hub's wizard is the authoritative source for downloads and always serves the correct, latest version of the Suite for your operating system automatically — no need to hunt for the right file.
</Info>

## 1.4 Install AgentOS

This is **Step 1 of 4** in the hub's Getting Started wizard.

**[AgentOS](/06-agent-os/index)** is the runtime that actually controls your screen — capturing screenshots, moving the mouse, pressing keys.

Follow the [AgentOS installation guide](/06-agent-os/installation/standalone) to download and install AgentOS for your operating system. In short:

1. Go to the [AgentOS installation page](/06-agent-os/installation/standalone) and follow the instructions for your OS.
2. Run the installer and follow the on-screen prompts.

<Tabs>
  <Tab title="Windows">
    If Windows says "Windows protected your PC", click **More info** → **Run anyway**. This is expected for software downloaded outside the Microsoft Store.
  </Tab>
  <Tab title="macOS">
    <Warning>
    **Required: Grant screen permissions.** AgentOS needs permission to see and control your screen. Without these, the agent cannot work.

    Open **System Settings → Privacy & Security → Accessibility** and toggle ON the switch next to **Terminal**. Then do the same under **Privacy & Security → Screen Recording**.

    If Terminal doesn't appear in the list, click **+**, navigate to Applications → Utilities → Terminal, and add it.
    </Warning>
  </Tab>
  <Tab title="Linux">
    <Warning>
    **Wayland:** AgentOS does not work on Wayland. To check, run:

    ```bash
    echo $XDG_SESSION_TYPE
    ```

    If it prints `wayland`, log out and select an Xorg session from your login screen before continuing.
    </Warning>
  </Tab>
</Tabs>

## 1.5 Create Your Access Token

This is **Step 2 of 4** in the hub's Getting Started wizard.

Access tokens authenticate your machine with AskUI's services. You also need your **Workspace ID** — it identifies your account. You'll put both into a credentials file in the next section.

<Steps>
  <Step title="Name your token">
    In the hub wizard (Step 2 of 4), type a name for your token — anything works, e.g. `my-laptop`.
  </Step>
  <Step title="Create it">
    Click **+ Create Token**.
  </Step>
  <Step title="Copy immediately">
    **Copy the token immediately.** The hub shows it once with the message "Copy this token now. You won't be able to see it again." Paste it somewhere safe — a note app or a text file on your Desktop — before clicking anything else.
  </Step>
  <Step title="Find your Workspace ID">
    Find your **Workspace ID** by looking at the URL bar in your browser. It's the long string after `/workspaces/`:

    `https://hub.askui.com/workspaces/0ebb7682-607f-49dd-884d-7c591f60738a`

    Copy that string and save it next to your token.
  </Step>
</Steps>

<Warning>
**Token shown only once.** If you navigate away before copying the token, you cannot retrieve it. You would need to go to **Access Tokens** in the sidebar, delete the token, and create a new one.
</Warning>

<Info>
**Concept:** This ID + token pattern is how almost every cloud API authenticates you. The Workspace ID says *who you are*. The token proves *you're authorized*. You'll encounter this same pattern with AWS, OpenAI, Stripe, Google Cloud, and virtually every SaaS tool with a developer API.
</Info>

## 1.6 Clone the Demo Project

This is **Step 3 of 4** in the hub's Getting Started wizard.

The hub shows a git clone command you can copy directly. It also offers a **Download ZIP** option if you prefer not to use Git.

Open your terminal and navigate to the folder you created in section 1.1:

```bash
cd ~/Desktop/askui-projects
```

Then clone the project using the command shown in the hub wizard (or copy from here):

```bash
git clone https://github.com/askui/AskUI-Demo-Project.git
```

When the download finishes, move into the project folder:

```bash
cd AskUI-Demo-Project
```

<Check>
Type `ls` and press Enter. You should see `main.py`, `requirements.txt`, `.env.template`, `tests/`, `prompts/`, `helpers/`, and `plans/` among the results.
</Check>

<Info>
**If you downloaded a ZIP instead:** Unzip the file, move the folder into `askui-projects` on your Desktop, then in your terminal run `cd ~/Desktop/askui-projects/AskUI-Demo-Project`.
</Info>

## 1.7 Install Dependencies & Set Up Your Credentials

Two quick steps to finish setup: install the Python libraries the project needs, then create your credentials file.

### Install dependencies

Make sure you're in your terminal inside the `AskUI-Demo-Project` folder. Copy and paste this command and press Enter:

<Tabs>
  <Tab title="Windows">
    ```bash
    pip install -r requirements.txt
    ```
  </Tab>
  <Tab title="macOS / Linux">
    ```bash
    pip3 install -r requirements.txt
    ```
  </Tab>
</Tabs>

You'll see packages downloading. When pip finishes, the last line says `Successfully installed ...` — including `askui` and `python-dotenv`.

### Set up your credentials file

The project includes `.env.template` — a ready-made template. You'll copy it and fill in the Workspace ID and Token you saved in section 1.5.

<Steps>
  <Step title="Create your own copy of the template">
    ```bash
    cp .env.template .env
    ```
  </Step>
  <Step title="Open the .env file in a text editor">
    <Tabs>
      <Tab title="Windows">
        ```bash
        notepad .env
        ```
      </Tab>
      <Tab title="macOS">
        ```bash
        open -e .env
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        nano .env
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Replace the placeholder values">
    Replace the placeholder values with your Workspace ID and Token from section 1.5. Delete the angle brackets too. The result should look like:

    ```
    ASKUI_WORKSPACE_ID=0ebb7682-607f-49dd-884d-7c591f60738a
    ASKUI_TOKEN=Dnp-0DDEQg0C4NPZKIRO
    ```
  </Step>
  <Step title="Save and close">
    Notepad/TextEdit: `Ctrl+S` (Mac: `Cmd+S`) then close. nano: `Ctrl+O`, Enter, `Ctrl+X`.
  </Step>
</Steps>

### What is a .env file?

A `.env` file stores secrets — API keys, tokens, passwords — in a local file that your code reads when it runs. The critical rule: **never share it, and never upload it to the internet.** The project's `.gitignore` file already tells Git to ignore `.env`, so your credentials stay private on your machine even if you share the rest of the project.

### Optional: Add an Anthropic API key

If you have an Anthropic API key from [console.anthropic.com](https://console.anthropic.com), open your `.env` file again and add a third line:

```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

This lets the agent use Claude models to reason about your screen. If you don't have one, the agent will use AskUI's default model configuration.
