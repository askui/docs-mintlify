Tools extend the capabilities of your agents beyond basic UI automation. They allow you to:

- Integrate external APIs and services
- Process data and perform calculations
- Manage files and system operations
- Handle complex business logic
- Create reusable functionality across different tasks

There are three ways to add tools to your agent: **pre-built tools** from the tool store, **MCP servers**, and **custom tools** you write yourself.

---

## Pre-Built Tools

The AskUI SDK includes a tool store with ready-to-use tools organized by category.

### How to use them

Pass tools either to a single `agent.act()` call or to the agent constructor:

```python
from askui import ComputerAgent
from askui.tools.store.computer import ComputerSaveScreenshotTool
from askui.tools.store.universal import PrintToConsoleTool

# Option 1: Pass to act()
with ComputerAgent() as agent:
    agent.act(
        "Take a screenshot and save it as demo.png, then print a status message",
        tools=[
            ComputerSaveScreenshotTool(base_dir="./screenshots"),
            PrintToConsoleTool()
        ]
    )

# Option 2: Pass to constructor (available for all act() calls)
with ComputerAgent(act_tools=[
    ComputerSaveScreenshotTool(base_dir="./screenshots"),
    PrintToConsoleTool()
]) as agent:
    agent.act("Take a screenshot and save it as demo.png, then print a status message")
```

### Tool categories

| Category | Import from | Works with | Examples |
| --- | --- | --- | --- |
| **Universal** | `askui.tools.store.universal` | Any agent | `PrintToConsoleTool()`, data processing, general utilities |
| **Computer** | `askui.tools.store.computer` | `ComputerAgent` | `ComputerSaveScreenshotTool(base_dir)`, window management |
| **Android** | `askui.tools.store.android` | `AndroidAgent` | Device info, app management, ADB commands |

## MCP Tools

The [Model Context Protocol (MCP)](https://modelcontextprotocol.io/docs/getting-started/intro) is a standardized way to provide tools to LLMs. AskUI supports MCP tools via [`fastmcp`](https://gofastmcp.com/getting-started/welcome).

```python
from fastmcp import Client
from fastmcp.mcp_config import MCPConfig, RemoteMCPServer

from askui import ComputerAgent
from askui.models.shared.tools import ToolCollection
from askui.tools.mcp.config import StdioMCPServer

mcp_config = MCPConfig(
    mcpServers={
        "my_server": StdioMCPServer(
            command="python", args=["-m", "my_mcp_server"]
        ),
    }
)

mcp_client = Client(mcp_config)
tools = ToolCollection(mcp_client=mcp_client)

with ComputerAgent() as agent:
    agent.act("Use the my_server tool to do something", tools=tools)
```

<Info>
Tool names are prefixed with the server name to avoid conflicts (e.g., `my_server_my_tool`). For different ways to construct clients, see the [fastmcp documentation](https://gofastmcp.com/clients/client).
</Info>

## Building Custom Tools

For functionality specific to your workflow, you can build your own tools. Each tool has three components:

1. **Class definition** — inherits from `Tool` with a descriptive docstring
2. **Constructor** — defines `name`, `description`, and `input_schema`
3. **Execution method** (`__call__`) — contains the actual logic

### Return types

A tool's `__call__` method can return:
- `str` — text output
- Numbers or other primitive values
- `PIL.Image.Image` — image output
- `None`
- A `list` or `tuple` containing any of the above

### Example: A greeting tool

This example is included in the Demo Project under `helpers/tools/greeting_tool.py`:

```python
from askui.models.shared.tools import Tool
from typing import Optional

class GreetingTool(Tool):
    """Creates personalized greeting messages with time-based customization."""

    def __init__(self):
        super().__init__(
            name="greeting_tool",
            description="Creates a personalized greeting message based on time of day and user preferences",
            input_schema={
                "type": "object",
                "properties": {
                    "name": {
                        "type": "string",
                        "description": "The name of the person to greet",
                        "minLength": 1
                    },
                    "time_of_day": {
                        "type": "string",
                        "description": "Time of day: morning, afternoon, or evening",
                        "enum": ["morning", "afternoon", "evening"]
                    },
                    "language": {
                        "type": "string",
                        "description": "Language for the greeting (optional). Default is english.",
                        "enum": ["english", "spanish", "french"],
                        "default": "english"
                    }
                },
                "required": ["name", "time_of_day"]
            }
        )

    def __call__(self, name: str, time_of_day: str, language: Optional[str] = "english") -> str:
        if not name or not name.strip():
            raise ValueError("Name cannot be empty")

        greetings = {
            "english": {"morning": "Good morning", "afternoon": "Good afternoon", "evening": "Good evening"},
            "spanish": {"morning": "Buenos dias", "afternoon": "Buenas tardes", "evening": "Buenas noches"},
            "french": {"morning": "Bonjour", "afternoon": "Bon apres-midi", "evening": "Bonsoir"},
        }

        base_greeting = greetings.get(language, greetings["english"])[time_of_day]
        return f"{base_greeting}, {name}! How are you today?"
```

### Using the tool

```python
from askui import ComputerAgent
from helpers.tools.greeting_tool import GreetingTool

with ComputerAgent() as agent:
    agent.act(
        "Greet John in the morning using Spanish",
        tools=[GreetingTool()],
    )
```

### Registering tools in the Demo Project

The Demo Project uses `helpers/get_tools.py` to register all custom tools in one place:

```python
from askui.models.shared.tools import Tool
from .tools.greeting_tool import GreetingTool

def get_agent_tools() -> list[Tool]:
    """Get the custom tools for the agent."""
    return [
        GreetingTool(),
    ]
```

To add a new tool: create a file in `helpers/tools/`, define your tool class, then add it to the list in `get_tools.py`.

### Tools with external dependencies

When your tool uses external services or APIs, store dependencies as underscore-prefixed private variables to prevent Pydantic validation errors:

```python
class ApiTool(Tool):
    """Calls an external API."""

    _client: Any  # Private — not validated by Pydantic

    def __init__(self, api_key: str):
        super().__init__(
            name="api_tool",
            description="Calls an external API to fetch data",
            input_schema={
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "The query to send"}
                },
                "required": ["query"]
            }
        )
        self._client = SomeApiClient(api_key=api_key)

    def __call__(self, query: str) -> str:
        return self._client.fetch(query)
```

### Tools that return images

Tools can return PIL Image objects alongside text:

```python
from PIL import Image
from askui.models.shared.tools import Tool

class ChartTool(Tool):
    """Generates a chart image."""

    def __init__(self):
        super().__init__(
            name="chart_tool",
            description="Generates a chart from data",
```

## Best Practices

| Area | Guidance |
| --- | --- |
| **Naming** | Use descriptive `snake_case` names reflecting purpose (e.g., `csv_data_processor`), not generic labels |
| **Schema** | Add `required` fields, use `enum` for constrained options, write detailed `description` for each parameter |
| **Errors** | Raise meaningful error messages — the agent has auto-correction and will retry with the error context |
| **Performance** | Keep tools focused and lightweight. Add timeouts for blocking operations. Cache expensive computations |
| **Documentation** | Write clear docstrings — they help both humans and the agent understand the tool's purpose |

For the full technical reference, see the [AskUI SDK tools documentation on GitHub](https://github.com/askui/python-sdk/blob/main/docs/07_tools.md).
