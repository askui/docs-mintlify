A properly configured development environment significantly improves productivity when working with AskUI. This guide covers setting up Visual Studio Code with the essential extensions and configurations for AskUI development.

## Prerequisites

- [Visual Studio Code](https://code.visualstudio.com/) installed
- Node.js (v18 or higher)
- AskUI project initialized

## Essential VSCode Extensions

### Required Extensions

Install these extensions for the best AskUI development experience:

**1. Jest Runner**
- **Extension ID**: `firsttris.vscode-jest-runner`
- **Purpose**: Run individual tests directly from the editor
- **Features**: 
  - Run/Debug buttons appear above test functions
  - Execute single tests without running the entire suite
  - Integrated debugging support

**2. ESLint**
- **Extension ID**: `dbaeumer.vscode-eslint`
- **Purpose**: Code quality and error detection
- **AskUI Benefits**:
  - Detects missing `.exec()` calls
  - Enforces coding standards
  - Shows AskUI-specific linting rules

**3. Live Preview**
- **Extension ID**: `ms-vscode.live-server`
- **Purpose**: View AskUI annotations and reports directly in VSCode
- **Features**:
  - Preview HTML annotation files
  - View Allure reports without external browser
  - Live reload for development

### Automatic Extension Installation

Create a `.vscode/extensions.json` file in your project root:

```json
{
    "recommendations": [
        "firsttris.vscode-jest-runner",
        "dbaeumer.vscode-eslint",
        "ms-vscode.live-server",
        "bradlc.vscode-tailwindcss",
        "ms-vscode.vscode-typescript-next"
    ]
}
```

When team members open the project, VSCode will prompt them to install the recommended extensions.

## VSCode Configuration

### Project Settings

Create a `.vscode/settings.json` file with AskUI-optimized settings:

```json
{
    "terminal.integrated.defaultProfile.windows": "Command Prompt",
    "terminal.integrated.profiles.windows": {
        "AskUI Shell": {
            "path": "cmd.exe",
            "args": ["/c", "askui-shell"]
        }
    },
    "eslint.validate": [
        "javascript",
        "typescript"
    ],
    "editor.codeActionsOnSave": {
        "source.fixAll.eslint": "explicit"
    },
    "typescript.preferences.importModuleSpecifier": "relative",
    "jest.autoRun": "off",
    "jest.showCoverageOnLoad": false,
    "files.exclude": {
        "**/node_modules": true,
        "**/allure-results": true,
        "**/allure-report": true,
        "**/.askui/cache": true
    },
    "search.exclude": {
        "**/node_modules": true,
        "**/allure-results": true,
        "**/allure-report": true
    }
}
```

### Key Configuration Benefits

**Terminal Integration**: Sets up AskUI Shell as a terminal profile
**ESLint Auto-fix**: Automatically fixes common issues on save
**File Exclusions**: Hides generated files from search and explorer
**Jest Configuration**: Optimizes Jest Runner behavior

## Development Workflow Setup

### Running Tests in VSCode

With Jest Runner installed, you'll see **Run** and **Debug** buttons above your test functions:

```typescript
// purchase-flow.test.ts
describe('Purchase Flow', () => {
    it('should complete standard user purchase', async () => {
        // Run | Debug buttons appear here â†‘
        await loginPage.login(testData.users.standard);
        await inventoryPage.addItemToCart('Sauce Labs Backpack');
        await checkoutPage.completeCheckout(testData.checkout.validInfo);
        
        await expect(confirmationPage.isDisplayed()).resolves.toBe(true);
    });
});
```

### Using the Integrated Terminal

1. **Open AskUI Shell Terminal**:
   - Press `Ctrl+Shift+P` (or `Cmd+Shift+P` on Mac)
   - Type "Terminal: Select Default Profile"
   - Choose "AskUI Shell"

2. **Common Commands**:
   ```bash
   # Start AskUI services
   askui-startcontroller
   
   # Run all tests
   askui-runproject
   
   # Run specific test file
   npx jest path/to/test.test.ts --config jest.config.ts
   ```

## Debugging Configuration

### VSCode Debug Configuration

Create a `.vscode/launch.json` file for debugging tests:

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Debug AskUI Test",
            "type": "node",
            "request": "launch",
            "program": "${workspaceFolder}/node_modules/.bin/jest",
            "args": [
                "--runInBand",
                "--no-cache",
                "--config",
                "jest.config.ts",
                "${relativeFile}"
            ],
            "env": {
                "NODE_ENV": "test"
            },
            "console": "integratedTerminal",
            "internalConsoleOptions": "neverOpen",
            "disableOptimisticBPs": true,
            "skipFiles": [
                "<node_internals>/**"
            ]
        },
        {
            "name": "Debug Current Test File",
            "type": "node",
            "request": "launch",
            "program": "${workspaceFolder}/node_modules/.bin/jest",
            "args": [
                "--runInBand",
                "--no-cache",
                "--config",
                "jest.config.ts",
                "${fileBasenameNoExtension}"
            ],
            "env": {
                "NODE_ENV": "test"
            },
            "console": "integratedTerminal"
        }
    ]
}
```

### Debugging Best Practices

1. **Set Breakpoints**: Click in the gutter next to line numbers
2. **Use Debug Console**: Inspect variables and execute AskUI commands
3. **Step Through Code**: Use F10 (step over) and F11 (step into)
4. **Watch Variables**: Add expressions to the Watch panel

## Troubleshooting Common Issues

### Jest Runner Not Working

If Jest Runner buttons don't appear:

1. **Check Jest Configuration**: Ensure `jest.config.ts` is properly configured
2. **Verify File Patterns**: Jest Runner looks for `*.test.ts` or `*.spec.ts` files
3. **Reload Window**: Sometimes VSCode needs a reload to recognize Jest tests

### ESLint Not Running

If ESLint isn't showing errors:

1. **Check ESLint Installation**: Ensure ESLint is installed in your project
2. **Verify Configuration**: Check `.eslintrc.json` exists and is valid
3. **Enable Auto-fix**: Add ESLint auto-fix on save to settings

### Terminal Issues

If AskUI Shell doesn't work in terminal:

1. **Check Path**: Ensure AskUI is properly installed and in PATH
2. **Use Full Path**: Specify full path to `askui-shell` executable
3. **Permission Issues**: On macOS/Linux, ensure execute permissions
